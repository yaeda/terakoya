# TERAKOYA 仕様

この文書は、TERAKOYA の現在のプロダクト挙動と運用前提を定める基準文書である。

## 1. 製品概要

TERAKOYA は、日本語学習向けの印刷用ワークシートをブラウザ上で生成するアプリケーションである。利用者は公開された Google スプレッドシートを読み込み、問題候補を絞り込み、出題をランダム選択し、画面プレビューを確認したうえで A4 ワークシートを印刷する。

- サーバー側に永続化を持たない、クライアントサイド完結のシングルページアプリケーションである
- 主な成果物は、画面内の対話型クイズではなく、印刷用ワークシートである
- 現在のデプロイ先は GitHub Pages の `/terakoya/` 配下である

## 2. 主な利用フロー

想定する主な用途は、漢字を中心とした読み書き練習用プリントの作成である。

基本的な利用手順は次のとおり。

1. ブラウザで TERAKOYA を開く
2. Google スプレッドシートの URL を入力または既定値のまま利用する
3. カテゴリと直近結果で問題候補を絞り込む
4. 最大 20 問をランダムに選ぶ
5. タイトル、答えの表示方法、読み書きモード、記入欄の表示を調整する
6. 拡大縮小されたプレビューを確認する
7. 印刷する

## 3. 画面構成

### Desktop

- 画面はワークシートのプレビュー領域と設定サイドバーに分かれる
- プレビューは左側に表示され、利用可能なビューポートへ自動縮尺される
- サイドバーには設定カード群と印刷ボタンが表示される

### Mobile

- メイン画面にはプレビューが表示される
- 設定はフローティングボタンから開く下部ドロワーに移る
- ドロワー内にはタブ化された設定群と印刷ボタンが表示される

### Print Mode

- 設定 UI やボタン類などの操作用 UI は非表示になる
- 印刷対象はワークシートのページのみである
- CSS の `@page` により、ページサイズは余白 0 の A4 縦とする

## 4. データソース仕様

### 対応ソース

- `https://docs.google.com/spreadsheets/d/<sheet-id>/...` に一致する Google スプレッドシート URL を受け付ける
- データ取得は、スプレッドシートの CSV エクスポートエンドポイントをクライアントから直接呼び出して行う
- 対象スプレッドシートは、アプリを実行するブラウザから公開閲覧できる必要がある

URL が想定する Google スプレッドシート形式に一致しない場合、アプリは空データとして扱い、0 件を読み込む。

### 行フォーマット

コメント行以外の各行は、列位置ベースで解釈する。

| 列 index | 項目 | 必須性 | 現在の挙動 |
| --- | --- | --- | --- |
| `0` | `id` | 推奨 | 各問題の小さな識別子として表示する |
| `1` | `question` | 必須 | 表示トークンへパースする。空の場合その行は無視する |
| `2` | `answer` | 任意 | その問題の答え一覧の先頭要素として使う |
| `3` | `category` | 任意 | カテゴリ絞り込みに使う |
| `4` | `lastDate` | 実質任意 | `Date` として解釈するが、現状 UI ロジックでは未使用 |
| `5` | `results` | 任意 | 直近結果ステータス算出のために 1 文字ずつ解釈する |

### 取り込み時の除外ルール

- 1 列目が `#` で始まる行はコメント行として無視する
- 2 列目の `question` が空の行は無視する
- 上記以外の厳密なスキーマ検証や、利用者向けの自動補正は現状行わない

### `results` 列の解釈

`results` は、学習結果の履歴文字列として扱う。

- 文字 `o` のみを正解とみなす
- `o` 以外の文字はすべて不正解として扱う
- 空または未設定の `results` は、履歴なしとみなす

アプリは `results` から `lastTwoResultStatus` を以下のように導出する。

| ステータス | 意味 |
| --- | --- |
| `zero` | 履歴が 0 件 |
| `less` | 履歴が 1 件 |
| `none` | 直近 2 件がどちらも不正解 |
| `one` | 直近 2 件のうち 1 件だけ正解 |
| `all` | 直近 2 件がどちらも正解 |

履歴が 3 件以上ある場合でも、この判定に使うのは末尾 2 件のみである。

## 5. 問題文の記法

`question` 列では、簡易的なインライン記法を利用できる。

### 通常テキスト

特殊記法に一致しない部分は、そのまま表示する。

### ルビ記法

対応形式:

- `((漢字::かんじ))`

現在の挙動:

- ルビ対象は、現在のパーサー規則に一致する漢字列である必要がある
- 表示時は HTML の `<ruby>` として描画する
- 現在認識するのは `((対象::ルビ))` 形式であり、旧形式の `漢字((かんじ))` は認識しない

### 穴埋め記法

対応形式:

- `[[漢字]]`
- `[[漢字::よみ]]`
- `{{漢字}}`
- `{{漢字::よみ}}`

現在の挙動:

- 穴埋め本文の `漢字` は、モードに応じて見せる文字または隠す答え本体として扱う
- 任意の `よみ` は、書きモードでは紙面ヒント、読みモードでは答え出力用の値として使う
- 実装上は区切り記号として `:` を 1 個以上受け付けるが、文書上の正式表記は `::` とする

## 6. 設定値と永続化

アプリはワークシート設定を `localStorage` に保存し、ページ再読み込み後も復元する。

| キー | 既定値 | 意味 |
| --- | --- | --- |
| `terakoya:data-source-url` | 組み込みの Google スプレッドシート URL | 現在のデータソース |
| `terakoya:selected-categories` | `undefined` | 選択中のカテゴリフィルタ |
| `terakoya:last-result-status` | `["zero", "less", "none", "one", "all"]` | 有効化された直近結果フィルタ |
| `terakoya:selected-index` | `[]` | 現在選ばれている問題 index |
| `terakoya:title` | `漢字テスト` | ワークシートタイトル |
| `terakoya:answer-type` | `["none"]` | 答え表示モード |
| `terakoya:read-write` | `"write"` | 読み書きモード |
| `terakoya:date-box-visibility` | `true` | 日付欄の表示有無 |
| `terakoya:name-box-visibility` | `true` | 名前欄の表示有無 |
| `terakoya:score-box-visibility` | `true` | 点数欄の表示有無 |

## 7. 出題選定ロジック

### カテゴリ絞り込み

- カテゴリ一覧は、取り込んだレコードの `category` を出現順で一意化して作る
- 利用可能なカテゴリが 0 件のとき、カテゴリ選択 UI は無効化される
- カテゴリ未選択時は、すべてのカテゴリを対象とする

### 直近結果絞り込み

- 利用者は 5 種類の導出ステータスを任意の組み合わせで有効化できる
- 既定では 5 種類すべてが有効である

### 候補集合

各レコードは、次の両方を満たすとき出題候補に入る。

- カテゴリ条件を満たす
- そのレコードの `lastTwoResultStatus` が有効化された直近結果ステータスに含まれる

### ランダム選出

- 出題ボタン押下時、現在の候補集合からランダムに問題を選ぶ
- 同一回の選出では重複を許さない
- 選出数の上限は 20 件である
- 候補が 20 件未満のときは、存在する件数の範囲で選ぶ
- 候補が 0 件のとき、出題ボタンは無効化される
- 出題ボタンのラベルには現在の候補件数を表示する

現状の実装では、手動の問題順制御、手動の個別選択、出題数の任意指定には対応しない。

## 8. ワークシートの描画仕様

### 問題数とレイアウト

- ワークシートは常に 10 枠または 20 枠で描画する
- 選択問題数が 10 以下のときは、縦長の大きい 10 枠レイアウトを使う
- 選択問題数が 11 から 20 のときは、2 段の 20 枠レイアウトを使う
- 選択数が不足する場合は、印刷レイアウト維持のため空枠で補完する

### 番号と向き

- 問題番号は `①` から `⑳` を使う
- 問題エリアは縦書きで表示する
- 問題は右上から左下へ向かって配置する

### タイトルと記入欄

- タイトルは用紙右側に縦向きで表示する
- 日付、名前、点数の記入欄は、タイトルの下に縦に並べて表示する
- 日付、名前、点数の各記入欄は独立して ON/OFF できる

## 9. 読み書きモード

ワークシートには 2 種類の出題モードがある。

### 書きモード

- 穴埋めトークンは記入スペースとして表示する
- 穴埋めトークンにヒントがある場合、そのヒントをカタカナで表示する
- ヒントがない場合、答え本文は見えないが幅だけ維持して空欄を作る
- 答え出力では、穴埋め本文そのものを答えとして使う

### 読みモード

- 穴埋めトークンには元の本文をそのまま表示する
- 答え出力では、穴埋めトークンのヒントを使う
- ヒントがない穴埋めトークンは、答え出力用の穴埋め答えには含まれない

## 10. 答え表示モード

アプリは 4 種類の答え表示モードを持つ。

### `none`

- 答えは印刷しない

### `front`

- 答え一覧を表面の問題一覧の下に印刷する

### `back`

- 印刷時のみ 2 ページ目を追加する
- 表面でもレイアウト維持のため答え欄の領域は確保するが、文字は見えない状態にする
- 裏面の答え一覧は、紙を折り返したときに答えが見える向きになるよう、ページ下部に回転配置する

### `code`

- 印刷時のみ 2 ページ目を追加する
- 2 ページ目には答え一覧の代わりに QR コードを配置する
- QR コードの内容は、`①答え/補助答え` のような行を改行連結したテキストである

### 答え一覧の構成

各問題の答え行は、次の順で組み立てる。

1. `answer` 列に明示値があれば先頭に入れる
2. 続けて、問題文内の穴埋めトークンから導出した答えを加える
3. それらを `/` で連結する

## 11. プレビューと印刷

- 画面上のプレビューは `ResizeObserver` を使って利用可能領域へ自動縮尺する
- プレビューで見えるのは主たるワークシートページである
- 裏面用の答え出力は print 専用であり、画面上では 2 枚目のプレビューとしては表示しない
- 印刷は各ボタンからブラウザの `print()` API を呼び出して開始する

## 12. 現在の制約と既知の問題

- 通信失敗、権限不足、壊れたスプレッドシート内容に対する専用のエラー UI は現状ない
- スプレッドシート URL が想定形式に合わない場合、エラーメッセージなしで 0 件扱いになる
- `lastDate` は読み込まれるが、現状は表示やフィルタには使わない
- 対応データソースは公開 Google Sheets の CSV のみであり、ローカルファイルや他バックエンドには対応しない
- モバイルブラウザでは、答え表示が `back` または `code` のとき 2 ページ目が印刷されないことがある。詳細は [known issues](./known-issues.md) を参照

## 13. 技術構成とデプロイ

- Frontend: React 19, TypeScript, Vite
- State: Jotai による設定保持、TanStack Query によるスプレッドシート読込
- UI: HeroUI とローカル UI ラッパー、Tailwind CSS
- Testing: 現状の自動テストは Vitest によるテキストパーサー中心
- Deployment: `main` への push を契機に GitHub Pages へ build/deploy する
